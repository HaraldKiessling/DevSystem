## 📝 Wie kann ich beitragen?

### Arten von Beiträgen

- 🐛 **Bug-Reports**: Fehler melden
- ✨ **Feature-Requests**: Neue Features vorschlagen
- 📖 **Dokumentation**: Docs verbessern
- 💻 **Code**: Features implementieren oder Bugs fixen
- 🧪 **Testing**: Tests schreiben oder QS durchführen

---

## 🌿 Branch-Strategie

### Branch-Typen

```
feature/[feature-name]     # Neue Features
docs/[dokumenttyp]-[thema]  # Dokumentations-Updates
hotfix/[problem]            # Dringende Fixes
```

### Workflow

1. Fork erstellen (bei externen Contributors)
2. Feature-Branch von `main` erstellen
3. Änderungen committen
4. Pull Request erstellen
5. Code Review abwarten
6. Nach Merge: Branch löschen

Siehe [`docs/strategies/branch-strategie.md`](docs/strategies/branch-strategie.md) für Details.

## 💬 Commit-Nachrichten

### Konventionen

```
<typ>: <kurze Beschreibung>

<detaillierte Beschreibung>

```

### Typen

- `feat`: Neues Feature
- `fix`: Bug-Fix
- `docs`: Dokumentation
- `refactor`: Code-Refactoring
- `test`: Tests
- `chore`: Maintenance
