# Projekt-Completion-Checkliste

## Vor attempt_completion Tool-Verwendung

**KRITISCH:** Bevor `attempt_completion` verwendet wird, MÜSSEN folgende Punkte validiert sein:

### Git-Status
- [ ] Alle Änderungen committed
- [ ] Kein "working tree dirty"
- [ ] **Main-Branch zu GitHub gepusht** ⚠️
- [ ] **Alle Tags zu GitHub gepusht** ⚠️
- [ ] origin/main ist up-to-date

### Dokumentation
- [ ] README.md aktualisiert (falls relevant)
- [ ] CHANGELOG.md aktualisiert (falls relevant)
- [ ] Alle Links validiert

### Code-Qualität
- [ ] Tests durchgeführt (falls relevant)
- [ ] Linting erfolgreich (falls relevant)
- [ ] Build erfolgreich (falls relevant)

---

**WICHTIG:** attempt_completion sollte erst verwendet werden, wenn das Projekt vollständig finalisiert ist, inklusive GitHub-Push!
