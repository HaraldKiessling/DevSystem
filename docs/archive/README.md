# DevSystem Dokumentations-Archiv

Dieses Verzeichnis enthält historische Dokumentation, die nicht mehr aktiv gepflegt wird, aber für Retrospektiven, Audits und historische Referenz aufbewahrt wird.

## 📂 Struktur

### `phases/`
Abgeschlossene Projekt-Phasen (Phase 1, Phase 2)
- Reports über erfolgreiche Deployments
- Merge-Summaries
- Phase-spezifische Status-Dokumente

### `test-results/`
Historische Test-Ergebnisse
- VPS-Tests (Caddy, code-server, Qdrant)
- E2E-Validation-Reports
- Phase-spezifische Test-Durchläufe

### `git-branch-cleanup/`
Branch-Cleanup-Historie und Troubleshooting
- Branch-Cleanup-Reports
- GitHub-Default-Branch-Anleitungen
- Git-Sync-Reports

### `concepts/`
Alte Versionen konsolidierter Konzepte
- Code-server-Konzept (Duplikate, Teildokumente)
- Testkonzept (alte Versionen)

### `retrospectives/`
Archivierte Lessons Learned und Retrospektiven
- Phase-spezifische Verbesserungsvorschläge

### `troubleshooting/`
Gelöste Problem-Dokumentationen
- Debug-Reports für behobene Probleme
- VPS-Korrekturen

### `reports/`
Abgeschlossene Status-Reports und Optimierungen

## 📋 Archivierungs-Policy

Dokumente werden archiviert, wenn:
- **Projekt-Phase abgeschlossen** ist (z.B. Phase 1/2)
- **Problem gelöst** wurde (Debug-Reports)
- **Duplikate konsolidiert** wurden (obsolete Versionen)
- **Tests/Validierungen abgeschlossen** sind
- **Reports veraltet** sind (>6 Monate ohne Updates)

## 🔍 Zugriff

Alle archivierten Dokumente:
- Bleiben im Git-Repository
- Sind über Git-Historie verfügbar
- Können bei Bedarf referenziert werden
- Werden nicht mehr aktiv gepflegt

Links zu archivierten Dokumenten sollten auf dieses Archiv-Verzeichnis zeigen.
